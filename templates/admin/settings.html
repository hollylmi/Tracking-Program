What's backed up:

  • All database records (projects, entries, employees, schedules, etc.)
  • SQLite: downloads the .db file
  • Railway/Postgres: runs pg_dump and downloads a .sql file
Uploaded files (photos, DXF files, documents) are not included in this download. On Railway, these are stored in ephemeral storage — see the Railway setup guide below for how to handle this.
Railway Setup Guide

Required environment variables on Railway:

  • SECRET_KEY — any long random string (e.g. 32+ random chars). Never share this.
  • DATABASE_URL — auto-set by Railway when you add a Postgres plugin. Don't change it.
  • PORT — auto-set by Railway. Don't change it.

Enable automatic Postgres backups:

  1. In your Railway project, click the Postgres plugin
  2. Go to Backups tab
  3. Enable Scheduled Backups (daily recommended)
  4. Railway keeps 7 days of history on the Hobby plan

Uploaded files on Railway:

Railway's filesystem resets on every deploy. To keep uploaded files permanently, you need an object storage bucket:

  • Cloudflare R2 — free for up to 10 GB, no egress fees (recommended)
  • AWS S3 — very cheap, industry standard
  • Backblaze B2 — cheap, S3-compatible
Email Help

Getting email working:

  1. Gmail: Go to your Google Account → Security → 2-Step Verification (must be on) → App Passwords → Create one for "Mail". Use that 16-character code as your password here.
  2. Outlook / Office 365: Use smtp.office365.com port 587 with your regular email and password (or your IT team may need to enable SMTP AUTH for your account).
  3. Other providers: Check your email provider's SMTP settings documentation.

PDF reports are generated on-demand — no settings needed for PDF download. Email settings are only required for sending via the report page.
